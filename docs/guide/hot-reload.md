# 热重载

MayJS 支持热重载，修改脚本文件后无需重启服务器即可生效。

## 使用方式

1. 开启文件监听（默认已开启）
2. 编辑 `.js` 文件并保存
3. 服务脚本自动重载，普通脚本下次 `/mayjs run` 时生效

## 手动重载

```
/mayjs reload-script            # 重载所有脚本
/mayjs reload-script <脚本名>   # 重载指定脚本
```

## 不同脚本类型的行为

| 脚本类型 | 文件变更后的行为 |
|---------|----------------|
| 普通脚本 | 下次 `/mayjs run` 执行新版本 |
| 服务脚本 | 自动卸载旧实例 → 加载新实例（资源自动清理和恢复） |
| 模块 | 下次 `require()` 加载新版本 |

::: tip
修改模块文件后，引用它的服务脚本需要重载才能拿到新版本。
:::

## 调试日志

开发阶段可开启详细日志辅助排查：

```yaml
script-engine:
  log-execution-result: true
  log-cache-operations: true
  log-watcher-events: true
```

生产环境建议关闭。

## 注意事项

- 防抖时间默认 500ms，可在 `config.yml` 中调整
- 新建子目录会自动纳入监听范围
- 删除脚本文件会触发对应服务脚本的卸载
